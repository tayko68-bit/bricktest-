El mensaje Invalid response format (Formato de respuesta no válido) nos está dando una pista clave: **el servidor no está devolviendo el JSON que esperábamos, sino otra cosa (seguramente un texto plano o un HTML de error).** Esto suele pasar por dos motivos principales en este punto: ### 1. El culpable más probable: El "follón" del Multipart Manual Construir los bytes del formulario a mano (Uint8Array) en el navegador es muy frágil. Si un solo carácter \r\n o el boundary no le gusta a la API de Brickognize, el servidor se confunde, rechaza la petición y devuelve un error de texto (o una página de Cloudflare) que tu código intenta leer como JSON, haciendo que explote. ### 2. El fantasma del CORS Como estás probando desde un entorno cliente (veo en la captura que dice boveda-react arriba, posiblemente un entorno local o un despliegue de prueba en el móvil), el navegador de iOS bloquea o altera las peticiones directas a dominios externos si no cumplen estrictamente las políticas de seguridad. ## Cómo solucionarlo ahora mismo Vamos a simplificar el código eliminando toda la ingeniería de bytes manual y dejando que el navegador haga su magia de forma nativa con FormData.
